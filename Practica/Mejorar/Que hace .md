# Улучшения программы

1. **Проверять ввод пользователя**

Избегайте сбоев программы, если пользователь вводит не число при выборе опции.

2. **Использовать более понятные имена переменных**

Например:  
`numeros` → `telefono`  
`registro` → `contacto`

Так код будет легче читать.

3. **Организовать код по функциям**

Программа станет чище, если каждая опция будет отдельной функцией:

- для добавления контактов  
- для отображения контактов  
- для их сохранения

Это предотвращает дублирование и улучшает поддержку кода.

4. **Закрывать файлы безопасным способом**

Использование блока `with` при открытии файлов предотвращает забывание о закрытии.

5. **Добавить более понятное меню**

Меню можно выводить в отдельной функции и переиспользовать.

6. **Проверять данные, которые вводит пользователь**

Например, убедиться, что email содержит «@», а телефон состоит из цифр.

7. **Добавить загрузку записной книжки при запуске**

Если файл `agenda.bin` уже существует, программа может загружать данные автоматически.

8. **Давать более понятные сообщения пользователю**

Исправить орфографические ошибки:

«Seleciona» → «Selecciona»  
«munero» → «número»  
«gurdado» → «guardado»

Это улучшит внешний вид программы.

9. **Разрешить удаление или редактирование контакта**

Добавление более продвинутых функций.

10. **Обрабатывать ошибки при сохранении**

Например, уведомлять пользователя, если файл не удалось записать.
